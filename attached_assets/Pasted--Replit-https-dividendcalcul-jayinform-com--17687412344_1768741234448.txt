너는 시니어 프론트엔드/풀스택 엔지니어다. 내가 운영 중인 Replit 프로젝트(https://dividendcalcul.jayinform.com)의 기존 코드베이스를 그대로 활용해 “배당 스노우볼 시뮬레이터(Dividend Snowball Simulator)” 페이지를 추가한다.

가장 중요한 목표:
- “스노우볼이 시작되는 시점(전환점)”을 시각적으로 매우 명확하게 보여준다.
- 과장/투자 권유 표현 없이, “가정 기반 시뮬레이션”임을 분명히 고지한다.
- 기존 사이트(배당 재투자 계산기, /goal 목표 배당금 계산기, /guide 사용 가이드)와 동일한 디자인/레이아웃/컴포넌트 규칙을 따른다.
- 새로운 프레임워크/디자인 시스템을 추가하지 않는다.

========================================================
A) 라우트/네비게이션/레이아웃
========================================================
1) 새 페이지 라우트
- Path: /snowball
- 메뉴(상단 네비게이션 또는 계산기 탭 영역)에 다음 항목을 상위 레벨로 추가:
  “스노우볼 시뮬레이터”
- 페이지 상단 구성은 기존 계산기들과 동일:
  - H1: 배당 스노우볼 시뮬레이터
  - 서브타이틀: “배당이 배당을 낳기 시작하는 전환점을 계산합니다(가정 기반 시뮬레이션).”
  - 서브타이틀 아래에 광고 자리 주석 추가:
    <!-- ADSENSE_SLOT_TOP -->
  - 그 아래에 계산기 탭 UI가 있다면 동일 위치에 탭을 포함(재투자/목표/가이드/스노우볼). 탭 UI가 없다면 네비게이션 링크로 충분.
- 페이지 하단(결과 섹션 아래)에 광고 자리 주석 추가:
  <!-- ADSENSE_SLOT_BOTTOM -->

2) SEO (기존 SEO 유틸/컴포넌트 재사용)
- Title: “배당 스노우볼 시뮬레이터 | 배당이 기여금을 넘는 시점 계산”
- Meta description: “월 추가 투자와 배당 재투자 가정 하에서, 연 배당금이 연간 추가 투자금을 초과하는 ‘스노우볼 전환점’이 언제 발생하는지 계산합니다.”

========================================================
B) 입력값(폼)
========================================================
기존 재투자 계산기 입력 UI/컴포넌트를 최대한 재사용한다(가능하면 동일한 state/validation 유틸 공유).

입력 필드 (기본값은 합리적으로 설정):
1) 초기 투자금 (KRW) - default 10,000,000
2) 예상 배당률 (연 %, gross) - default 4.0
3) 배당 성장률 (연 %, optional) - default 0.0
4) 월 추가 투자금 (KRW) - default 500,000
5) 투자 기간 (년) - default 15
6) 재투자 여부 (toggle) - default ON

검증(인라인, 한국어):
- 금액/기간 음수 금지
- 배당률 > 0
- 투자 기간은 정수 1~60 범위로 제한(너무 큰 계산 방지)
- 유효하지 않으면 결과 숨기고 오류 안내

표기:
- KRW 천 단위 콤마
- %는 소수 1자리

========================================================
C) 핵심 계산 정의(정확성/투명성 최우선)
========================================================
이 페이지는 “투자 권유”가 아니라 “가정 기반 시뮬레이션”임을 UI에 명확히 고지한다.
세금/수수료/환율/가격 변동(자본손익)은 반영하지 않는다고 명시한다(현재 구현 기준과 일치해야 함).

연 단위 시뮬레이션(단순 모델):
- year = 1..N
- portfolioStart = 이전 해 portfolioEnd
- contributionAnnual = monthlyContribution * 12
- annualDividend(year) 계산 방식은 한 가지로 고정하고, ‘가정’ 섹션에 문장으로 명시:

[권장 방식]
- baseYield는 고정.
- dividend growth는 배당금(지급액)에 적용:
  annualDividend(year) = portfolioStart * (yield/100) * (1 + growth/100)^(year-1)

portfolioEnd 업데이트:
- if reinvest ON:
  portfolioEnd = portfolioStart + contributionAnnual + annualDividend(year)
- if reinvest OFF:
  portfolioEnd = portfolioStart + contributionAnnual
  (배당금은 현금으로 수령한다고 가정; 포트폴리오에는 더하지 않음)

Snowball 전환점 정의(가장 중요):
- annualDividend(year) >= contributionAnnual
- 위 조건을 최초로 만족하는 year를 snowballYear로 지정
- 기간 N 내에 만족하지 못하면 “기간 내 전환점 없음”으로 표시

추가로 2개의 보조 지표를 계산해 시각화에 활용:
- dividendToContributionRatio(year) = annualDividend(year) / contributionAnnual
- netGap(year) = annualDividend(year) - contributionAnnual

========================================================
D) 결과 UI(시각적으로 “전환점”이 한눈에 보이게)
========================================================
가장 중요한 것은 “스노우볼 전환점”을 ‘한 화면에서’ 직관적으로 보여주는 것.
기존 사이트와 동일한 카드/타이포 규칙을 사용.

1) 최상단 핵심 배지/카드(필수)
- 제목: “스노우볼 전환점”
- 내용:
  - 전환점이 존재하면: “{snowballYear}년 차부터 연 배당금이 연간 추가 투자금(=월 추가 투자금×12)을 초과합니다.”
  - 존재하지 않으면: “설정한 투자 기간({N}년) 내에는 연 배당금이 연간 추가 투자금을 초과하지 않습니다.”
- 보조 텍스트(항상):
  - “본 결과는 입력한 가정에 따른 시뮬레이션이며, 실제 결과와 다를 수 있습니다.”

2) 비교 카드 2개(필수, 나란히)
- 카드 A: “연간 추가 투자금” = contributionAnnual (KRW)
- 카드 B: “해당 연도 연 배당금” = annualDividend(snowballYear 또는 마지막 해) (KRW)
- 전환점이 있을 때는 카드 B에 “초과” 상태를 시각적으로 표시(테두리/배지 등. 색 지정은 기존 테마 규칙을 따를 것)

3) ‘전환점 타임라인’ 시각화(권장)
기존 프로젝트에 차트 라이브러리가 이미 있다면 재사용하고, 없다면 새 라이브러리 추가는 하지 말 것.
대신 CSS 기반의 간단한 시각화로 구현 가능:
- 1..N까지 가로로 나열된 바/점(연도 표시)
- snowballYear 지점에 “START” 또는 “전환점” 라벨
- 전환점이 없으면 마지막에 “Not yet” 표시

4) 연도별 테이블(필수)
표 컬럼:
- 연도
- 포트폴리오 가치(연말)
- 연 배당금
- 연간 추가 투자금
- 배당/기여 비율(배)
- 차이(배당-기여, KRW)

UI 요구:
- 모바일에서 가로 스크롤 허용
- 전환점 연도 행을 강조 표시(기존 스타일로)

5) 간단한 인사이트 박스(권장, 과장 금지)
예:
- “월 추가 투자금이 커질수록 전환점이 늦어질 수 있습니다(기여 기준이 커지기 때문).”
- “배당률/성장률 가정이 달라지면 전환점도 달라집니다.”
단정/보장 표현 금지.

6) 가정/Disclaimer 섹션(필수, 접기 가능)
다음 문구를 포함:
- “배당률/배당금은 고정이 아니며 변동될 수 있습니다.”
- “세금, 수수료, 환율, 주가 변동에 따른 자본손익은 반영하지 않았습니다.”
- “본 도구는 투자 조언이 아닌, 의사결정 보조를 위한 시뮬레이션입니다.”
